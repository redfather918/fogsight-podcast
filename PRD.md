## 1. 产品概述

### 1.1 一句话定义

用户上传 PDF 文档，系统自动生成「动画讲解 + 双人播客对谈」的短视频，一站式完成从文档到视频的转化。

### 1.2 背景与动机

- **知识传播痛点**：大量优质内容以 PDF 形式存在（论文、报告、教材），阅读门槛高、传播效率低。
- **视频化趋势**：短视频/播客是当前最高效的知识传播载体，但制作成本高、周期长。
- **技术成熟度**：LLM 内容生成 + TTS 语音合成 + 动画引擎三者已足够成熟，可以串联成自动化流水线。
- **已有验证**：Prototype 已跑通完整链路（DeepSeek 动画生成 + 火山引擎双人播客 TTS + 音画同步注入），产出可用。

### 1.3 产品定位

| 维度 | 描述 |
|------|------|
| 目标形态 | Web 应用（浏览器访问） |
| 核心价值 | PDF → 视频，零制作成本，3 分钟出片 |
| 差异化 | 双人播客对谈（非单调朗读）+ 动画可视化讲解 |
| 非目标 | 不做视频编辑器、不做素材库、不做社交分发 |

## 2. 目标用户

### 2.1 用户画像

| 画像 | 场景 | 频率 |
|------|------|------|
| **知识创作者** | 把论文/技术文档转成科普视频发抖音/B站 | 每周 2-5 次 |
| **教育工作者** | 把教材章节转成教学动画视频 | 每周 1-3 次 |
| **企业培训** | 把产品文档/操作手册转成培训视频 | 每月 5-10 次 |
| **个人学习** | 把长 PDF 转成音频+动画方便碎片化学习 | 不定期 |

### 2.2 用户故事

> **作为**一名知识创作者，  
> **我希望**上传一篇论文 PDF，系统自动生成 1-3 分钟的动画讲解视频，  
> **以便**我能快速在社交媒体发布科普内容，不用自己写脚本、配音、做动画。

## 3. 功能需求

### 3.1 功能列表

| ID | 功能 | 优先级 | 描述 |
|----|------|--------|------|
| F01 | PDF 上传 | P0 | 支持 PDF 文件上传，限制大小（≤50MB），限制页数（≤100 页） |
| F02 | 内容提取 | P0 | 从 PDF 中提取文本内容，支持中英文 |
| F03 | 主题识别 | P0 | 自动识别 PDF 核心主题，生成视频标题 |
| F04 | 脚本生成 | P0 | LLM 将 PDF 内容转化为双人播客对谈脚本 |
| F05 | 动画生成 | P0 | 基于脚本生成动画 HTML（Canvas 动画 + 双语字幕） |
| F06 | 播客音频 | P0 | 火山引擎双人播客 TTS 生成对谈音频 |
| F07 | 音画同步 | P0 | 将音频时长注入动画，实现自动同步播放 |
| F08 | 视频录制 | P0 | Puppeteer 录屏 + FFmpeg 合成最终 MP4 |
| F09 | 进度展示 | P0 | 实时显示处理进度（WebSocket 推送） |
| F10 | 视频预览 | P0 | 在网页中预览生成的视频 |
| F11 | 视频下载 | P0 | 下载 MP4 文件 |
| F12 | 历史记录 | P1 | 查看历史生成记录，重新下载 |
| F13 | 参数配置 | P1 | 可调参数：视频时长、语速、动画风格、音色选择 |
| F14 | 多格式导出 | P2 | 支持 9:16 竖屏 / 16:9 横屏切换 |
| F15 | 批量处理 | P2 | 批量上传 PDF，队列处理 |

### 3.2 核心流程

```
用户上传 PDF
    │
    ▼
┌─────────────┐
│  PDF 解析    │  提取文本，识别主题
└─────┬───────┘
      ▼
┌─────────────┐
│  脚本生成    │  LLM → 双人播客对谈脚本 + 旁白数据
└─────┬───────┘
      ▼
┌─────────────┐
│  动画生成    │  LLM → Canvas 动画 HTML（含字幕+音频接口）
└─────┬───────┘
      ▼
┌─────────────┐
│  音频生成    │  火山引擎 TTS → 双人播客 MP3
└─────┬───────┘
      ▼
┌─────────────┐
│  音画同步    │  注入音频路径 + 时长到 HTML
└─────┬───────┘
      ▼
┌─────────────┐
│  视频录制    │  Puppeteer 录屏 → WebM
└─────┬───────┘
      ▼
┌─────────────┐
│  视频合成    │  FFmpeg → 最终 MP4（视频+音频）
└─────┬───────┘
      ▼
  预览 + 下载
```

### 3.3 处理阶段与进度

| 阶段 | 名称 | 预估耗时 | 进度区间 |
|------|------|----------|----------|
| 1 | PDF 解析 | 2-5s | 0-10% |
| 2 | 脚本生成 | 10-20s | 10-25% |
| 3 | 动画生成 | 15-30s | 25-45% |
| 4 | 音频生成 | 30-90s | 45-70% |
| 5 | 音画同步 | 1-2s | 70-72% |
| 6 | 视频录制 | = 动画时长 | 72-92% |
| 7 | 视频合成 | 5-15s | 92-100% |

**总预估**：1-3 分钟（取决于内容长度和目标时长）

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| **性能** | 单任务端到端 ≤ 3 分钟（60s 目标视频）；并发支持 5 个任务 |
| **可用性** | 服务可用率 ≥ 99%；任务失败可重试 |
| **安全** | PDF 文件不持久化存储（处理完即删）；API 密钥不明文暴露 |
| **兼容** | 支持 Chrome / Edge / Safari 最新版；移动端适配 |
| **限制** | 单文件 ≤ 50MB；页数 ≤ 100 页；输出视频 ≤ 5 分钟 |
| **许可** | Fogsight 引擎 CC BY-NC-ND 4.0（非商业）；火山引擎按量付费 |
